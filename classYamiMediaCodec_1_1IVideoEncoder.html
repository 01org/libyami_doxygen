Detailed Description

Abstract video encoding interface of libyami it is the interface with client. they are not thread safe. getOutput() can be called in one (and only one) separated thread, though IVideoEncoder APIs are not fully thread safe.

virtual Encode_Status YamiMediaCodec::IVideoEncoder::getOutput ( VideoEncOutputBuffer *  outBuffer,
bool  withWait = false 
)
pure virtual

return one frame encoded data to client; when withWait is false, ENCODE_BUFFER_NO_MORE will be returned if there is no available frame.
when withWait is true, function call is block until there is one frame available.
typically, getOutput() is called in a separate thread (than encoding thread), this thread sleeps when there is no output available when withWait is true.

param [in/out] outBuffer a VideoEncOutputBuffer of one frame encoded data param [in/out] when there is no output data available, wait or not
